# 实验室物资管理工具

这是一个基于 React + Vite 搭建的实验室试剂 / 耗材管理前端原型，用于解决课题组物资信息分散、重复购买、查找困难、常用耗材补充不及时等问题。

项目在线预览：  
https://840985964-gif.github.io/lab-inventory-manager/

---

## 项目背景

在课题组实际使用场景中，不同课题成员会自行购买不同试剂和耗材，但这些信息通常分散在个人记录、冰箱贴纸或群消息中，导致其他成员难以及时了解组内已有物资情况。

常见问题包括：

- 不知道组里是否已有某种试剂或耗材，导致重复购买；
- 试剂分散存放在不同冰箱和试剂柜中，查找成本高；
- 常用耗材用完前不能及时发现，影响实验进度；
- 传统纸质台账流动性差，不便于更新和导出。

因此，本项目尝试通过一个轻量化前端工具，帮助实验室成员快速录入、查询、修改、删除和导出试剂 / 耗材信息。

## 当前版本功能

### 1. 首页分类入口

首页包含两个主要入口：

- 试剂库
- 耗材库

用户可以根据物资类型进入对应管理页面。

---

### 2. 试剂库存放点管理

考虑到实验室试剂通常分散存放在多个冰箱和试剂柜中，当前版本将试剂库设计为两层结构：

    进入试剂库 → 创建 / 选择具体存放点 → 进入该存放点 → 管理其中的试剂信息

支持创建不同试剂存放点，例如：

- 1号冰箱
- 2号冰箱
- 有机试剂柜
- 常温试剂柜

每个存放点支持：

- 新增存放点
- 编辑存放点
- 删除存放点
- 查看当前存放点内的试剂数量
- 进入存放点管理具体试剂

---

### 3. 试剂信息管理

进入具体存放点后，可以管理该位置下的试剂信息。

试剂字段包括：

- 名称
- 规格
- 数量
- 具体位置
- 开启日期
- 备注

支持功能：

- 新增试剂
- 编辑试剂
- 删除试剂
- 搜索试剂
- CSV 导出
- localStorage 本地保存

---

### 4. 耗材信息管理

耗材库保持轻量化结构，直接进行耗材录入与管理。

耗材字段包括：

- 名称
- 规格
- 数量
- 存放位置
- 购买日期
- 备注

支持功能：

- 新增耗材
- 编辑耗材
- 删除耗材
- 搜索耗材
- CSV 导出
- localStorage 本地保存

---

### 5. CSV 导出

当前版本支持将物资清单导出为 CSV 文件。

试剂库导出文件名格式：

    YYYY-MM-DD_存放点名称_试剂清单.csv

示例：

    2026-05-27_1号冰箱_试剂清单.csv

耗材库导出文件名格式：

    YYYY-MM-DD_耗材清单.csv

示例：

    2026-05-27_耗材清单.csv

---

### 6. 默认示例数据

为了方便在线展示，项目在首次打开且 localStorage 为空时，会初始化部分示例数据。

示例数据支持：

- 展示
- 搜索
- 编辑
- 删除
- CSV 导出

用户后续新增、编辑或删除数据后，刷新页面不会被默认示例数据覆盖。

## 本地启动方式

进入前端项目目录：

    cd frontend

安装依赖：

    npm install

启动本地开发环境：

    npm run dev

构建生产版本：

    npm run build

部署到 GitHub Pages：

    npm run deploy

## 当前版本边界

当前项目是一个纯前端 MVP，数据通过 localStorage 保存在用户本地浏览器中。

因此：

- 不同用户之间数据不会自动同步；
- 换设备或换浏览器后无法看到原浏览器中的数据；
- 清除浏览器缓存可能导致本地数据丢失；
- 暂不支持账号登录、权限管理和多人协作。

## 项目说明

这个项目是一个个人练习型 vibecoding 项目。开发过程中，我先基于课题组真实物资管理痛点进行需求拆解，再使用 ChatGPT 和 Trae 辅助完成原型设计、字段调整、功能迭代和部署。

项目经历了从“类商业实验室管理系统”到“课题组轻量化物资管理工具”的收敛过程。最终版本更强调低填写成本、清晰存放层级和可快速上手的使用体验。
